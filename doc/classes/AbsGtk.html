Methods
Screen:add_label (id, label)
adds a label to the screen widgets table.

Parameters:

  • id the id to reference the widget later on
  • label the label itself that will be written
Screen:add_button (id, label[, tooltip[, callback]])
creates a button and adds it to the screen widgets table.

Parameters:

  • id the id to reference the widget later on
  • label the label that will be written over the button
  • tooltip a tooltip to the button (optional)
  • callback a callback function to the button (optional)
Screen:create_button_box (id, labels[, tooltip[, callback]])
creates a buttonset and adds it to the screen widgets table.

Parameters:

  • id the id to reference the widget later on
  • labels the labels that will be written over the buttons
  • tooltip a tooltip to the buttons (optional)
  • callback a callback function to the buttons (optional)
Screen:create_combobox (id, labels[, default_value[, tooltip[, callback]]])
creates a dropdown menu and adds it to the screen widgets table.

Parameters:

  • id the id to reference the widget later on
  • labels the labels that will be written on the rows
  • default_value the index of the starting row (optional)
  • tooltip a tooltip to the combobox (optional)
  • callback a callback function to the row (optional)
Screen:add_image (id, path[, dimensions[, tooltip]])
creates an image widget and adds it to the screen widgets table.

Parameters:

  • id the id to reference the widget later on
  • path the path of the image file
  • dimensions a table with the dimensions to resize the image (optional)
  • tooltip a tooltip to the image (optional)
Screen:add_text_input (id[, label[, visibility[, default_value[, tooltip[, callback]]]]])
creates a text input field and adds it to the screen widgets table.

Parameters:

  • id the id to reference the widget later on
  • label a label that precedes the field (optional)
  • visibility passed by abstk module, client call a different function depending on whether it wants, a common field or a password one (optional)
  • default_value a placeholder (optional)
  • tooltip a tooltip to the text input field (optional)
  • callback a callback function to the field (optional)

table that represents a Wizard.

Fields:

  • assistant a table that holds every aspect of the wizard window, such as its title, dimensions and etc.
  • pages a table where all the screens will be stored
